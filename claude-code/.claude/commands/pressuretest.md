# /pressuretest

Use this to challenge a plan before committing to it.

Plan or idea:
$ARGUMENTS

Goal:
Expose weak assumptions before implementation.

Process:
1. Ask hard questions.
2. Challenge assumptions.
3. Identify risks and failure modes.
4. Identify hidden costs and tradeoffs.
5. Suggest the smallest viable alternative.
6. Do not implement code.

Output:
- Strongest objections
- Unanswered questions
- Failure modes
- Simpler alternative
- Go / no-go recommendation
