# /onboard

Use this at the start of work in an unfamiliar repository.

Repository or area:
$ARGUMENTS

Required workflow:
1. Read the top-level docs and agent instructions.
2. Identify stack, package manager, verification commands, and deployment shape.
3. Inspect repo structure without reading every file.
4. Find the key entry points for the requested work.
5. Run read-only status/version checks where useful.
6. Summarize what matters for the current task.

Output:
- Stack
- Entry points
- Important commands
- Constraints
- Risks
- Recommended next workflow
