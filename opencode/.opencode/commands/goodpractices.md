# /goodpractices

Use this to improve a new or existing project without boiling the ocean.

Target:
$ARGUMENTS

Review:
- Repository structure
- Build/test/lint/typecheck commands
- Dependency hygiene
- Docker and CI
- Documentation
- Agent instructions
- Security defaults
- Developer onboarding

Output:
- Top 5 improvements
- Why each matters
- Smallest first step
- Risk
- Verification command

Do not refactor code unless explicitly asked.
