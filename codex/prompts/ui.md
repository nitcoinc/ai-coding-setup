# /ui

Use this before creating or changing frontend UI.

UI task:
$ARGUMENTS

Required workflow:
1. Identify the user, workflow, device sizes, and emotional tone.
2. Reuse the existing design system if one exists.
3. If no design system exists, choose a clear visual direction before coding.
4. Avoid generic layouts, decorative clutter, and text that explains the app instead of making it usable.
5. Define responsive behavior and empty/loading/error states.
6. Verify desktop and mobile with Playwright.
7. Check text overflow, overlap, tap targets, keyboard flow, and accessibility basics.
8. Add or update visual/e2e tests when the UI is core to the feature.

Output:
- UX intent
- Layout approach
- States
- Responsive checks
- Accessibility checks
- Verification plan
